## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Observer-unit conversion | Any metres→render conversion must be `metres / ship.LocalPos.Scale * factor`. Using `metres * factor` alone is wrong at any non-unity scale (canonical Phase-5 lesson; drove multiple marker-placement iterations in Phase 6). | advisory | Cite `05-03-SUMMARY.md §Observer-Unit Render Conversion`; the marker now sidesteps it entirely by using angular size (a scale-independent ratio) — follow that pattern for future render work. |
| Read-only HUD/renderers (D-53) | Hud, TargetSelectorPanel, and TargetMarkerRenderer must never mutate `GameObjects` / `LocalPos` / call `TranslatePos`. Selection flows only through `Hud.SetTargetIndex`. | advisory | Verified by grep in 06-VERIFICATION.md; keep any new renderer a pure read-only consumer. |
| Cross-frame math | Never `body.LocalPos.ToDouble3()` or cross-scale `UniVec3` subtraction for a cross-space body → catastrophic cancellation at ~1e30 m. | advisory | Use `UniMath.RelativeMetres` / `Distance` / `RelativePosition` (LCA path); CLAUDE.md §Position Math. |
