CANONICAL observer-unit conversion (Phase 5, 05-03-SUMMARY, MUST follow — D-51):
- Position (cross-space): relMetres = UniMath.RelativeMetres(ship, target, objs);
  obsFactor = factor / ship.LocalPos.Scale; renderPos = relMetres.{X,Y,Z} * obsFactor.
  This mirrors WorldRenderer.ComputeStarRenderPosFromHierarchy lines 395-406.
- Radius: r = (rawRadiusMeters / ship.LocalPos.Scale) * factor, mirroring WorldRenderer.RenderBodyAt lines 454-456.
- NEVER metres * factor alone — that is always wrong at non-unity scale (the 05-03 depth lesson).

On-screen pixel radius (for the D-52 floor) — port from the removed Hud.UpdateTargetCircle analytic
formula (it lived at Hud.cs lines 418-433 before 06-01 removed it; the formula is reproduced in
06-PATTERNS.md): depth = -camLocal.Z; pixelRadius = refExtent * (renderRadius / depth) / tan(fovY/2).
The D-52 floor is applied AFTER projecting to pixels: if the projected pixel radius < MinMarkerRadius,
hold the marker at the floor (scale the mesh so its projected radius equals the floor, OR draw a
fixed-size 2D fallback reticle — implementer's choice, must stay a small round outline).
</interface_context>
