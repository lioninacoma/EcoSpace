<objective>
Replace Phase-7's exponential warp-approach timing with a closed-form symmetric smoothstep
**trapezoidal velocity profile** so warp arrival time matches the player-selected travel
time `T_sel` exactly across all scales, and the ship decelerates *into* the target SOI
instead of tunnelling past it. Drop the `WarpMaxSpeed` cap (supersedes Phase-7 D-07),
expose the accel-fraction `f` as a tuning knob, drive warp by elapsed time, and harden
`EngageWarp` against an invalid target (WR-03).

Purpose: Exact, reproducible, scale-invariant warp timing is the headline user requirement
for Phase 8. This plan delivers the core math + controller rewrite that every other Phase-8
change depends on.
Output: A pure `WarpMotionProfile` struct with unit tests, plus the rewritten warp drive in
`FlightController` (EngageWarp solve, _WarpProcess elapsed-time drive, new `f` export,
`WarpMaxSpeed` removed, WR-03 safe-abort gate).
</objective>

authentication, no persisted user data, and no external input beyond local keyboard/mouse**.
