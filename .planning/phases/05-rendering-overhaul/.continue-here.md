# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure this session.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: Metres→render units conversion — any metres value converted to render/world units in this engine MUST go through `metres / ship.LocalPos.Scale * StarRenderFactor` (the observer-unit basis), NEVER `metres * factor` — structural mitigation: reuse `LuminousBodyDescriptor.RenderDistance` (already computed correctly in the builder) instead of re-deriving render distances in a drawer.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Bare `metres * factor` render conversion | The 05-03 PSF depth gate computed star render-depth as `DistanceMeters * StarRenderFactor`, omitting the `/ ship.LocalPos.Scale` observer-unit divisor. In Planet space (scale=1e-4) this made the star depth 1e4× too small (1496 vs 1.496e7 render units), so foreground-planet occlusion only worked at very close range ("growing circle"). | blocking | Always convert via `metres / ship.LocalPos.Scale * factor` (the conversion WorldRenderer uses at all 4 call sites). Prefer reading the pre-computed `LuminousBodyDescriptor.RenderDistance` rather than re-deriving. |
| Drawer self-resolving the world | LuminousPassRenderer (a Camera3D child with no `WorldPath`) tried to resolve TestSetup/ship itself, got null, and a `if (ship==null){star_count=0;return;}` guard zeroed the PSF every frame. | advisory | Drawers consume the descriptor (built by LuminousDescriptorBuilder, which reliably has the ship). Do not add per-drawer world/ship resolution; put any ship-dependent math in the builder and expose it on the descriptor (D-02). |
| Manual clip-space billboard MultiMesh (StarPointRenderer) | Exhaustively tried and abandoned in the old (deleted) Phase 5; structurally fragile against the engine. | blocking | Distant bodies live in the sky shader (D-07); near-star glow is the depth-aware post-process PSF. Never reintroduce a manual billboard MultiMesh. |
