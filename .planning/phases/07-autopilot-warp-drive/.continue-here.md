# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure this session.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: HSlider-focus-capture — HSlider accepts keyboard focus by default; `ui_accept` (Enter) is consumed by the slider before reaching `_UnhandledInput`. Always set `FocusMode = FocusModeEnum.None` on any HSlider inside a `_UnhandledInput`-driven panel. Structural mitigation: the fix is already committed (7569a2e) — verify `FocusModeEnum.None` is still present in `WarpConfirmationScreen.cs` before touching that file.
- [ ] CONSTRAINT: look-around-cursor-bleed — `_cursor` accumulated in `_Input` regardless of whether `look_around` (Left Alt) was held. On Alt release, the stale cursor immediately steered the ship. Fix: separate `_rawMouseDelta` field captures raw motion every frame; `_Input` skips writing to `_cursor` while Alt is held; `_cursor` is zeroed each frame during look-around. Structural mitigation: fix is in commit 7569a2e — do NOT revert or simplify this dual-field pattern.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| HSlider-focus-capture | HSlider's default FocusMode captures `ui_accept` (Enter), silently swallowing the keypress before `_UnhandledInput` fires — warp engage appeared broken. | blocking | Always set `FocusMode = FocusModeEnum.None` on HSlider in any panel whose engagement is driven by `_UnhandledInput`. |
| look-around-cursor-bleed | `_cursor` was shared between look-around camera offset and ship steering. Holding Alt panned the camera using the accumulated steering cursor, then releasing Alt applied that same delta as a steering impulse. | blocking | Use a dedicated `_rawMouseDelta` field for look-around; gate `_cursor` accumulation on `!look_around` being held; zero `_cursor` each look-around frame. |
