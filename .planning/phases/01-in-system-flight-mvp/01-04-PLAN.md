## Phase Goal

**As a** pilot, **I want to** fly my ship through a star system and approach dithered planets and a star while reading my speed and surroundings on a retro HUD, **so that** I can navigate a true 1:1 cosmos that stays correct and feels good at every scale.

## MVP Slice — Minimal Retro HUD (and TRV-01 integration milestone)

This final slice completes the HUD: the skeleton's raw-m/s number becomes an adaptive-unit readout, a context label names where you are and what's nearest, and a cycle-able target readout (with an off-screen direction marker so the 1:1 world stays findable) lets you pick and aim at bodies. After this plan, a real user can do the full Phase-1 loop — fly, approach dithered bodies, and read speed + context + target — which composes the TRV-01 in-system-flight milestone. This is the integration point where all five slices meet.

<threat_model>
## Trust Boundaries

Single-player offline; no network/auth/persistence. The HUD is a pure READ-ONLY consumer of simulation + flight state — it has no mutation path, so its only failure modes are display errors or a read-time null/index dereference, not state corruption.

| Boundary | Description |
|----------|-------------|
| `GameObjects` index references → HUD reads | HUD scans `ChildIndices` and indexes `GameObjects` for nearest-body / target; stale or null indices could be dereferenced |
| target `_targetIndex` across SOI transition | A cycled target index can become invalid when the ship changes parent space |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering / crash | HUD dereference of a null/stale `GameObjects` slot during nearest/target scan | mitigate | Route HUD lookups through the null-safe `Get(i)` / `(uint)i < (uint)Count` guard and null-skip (Tasks 1-2), consistent with STAB-01 hardening |
| T-04-02 | Tampering / crash | `_targetIndex` pointing at a sibling that left the current space after an SOI transition | mitigate | Clamp/reset `_targetIndex` to a valid current-space sibling each frame (Task 2) |
| T-04-03 | Denial of Service (visual) | divide-by-zero / NaN in adaptive-unit formatting at zero speed/distance | mitigate | `FormatSpeed`/`FormatDistance` branch on magnitude bands before dividing; zero is handled by the lowest (m/s) band (Task 1) |

No network, auth, session, secrets, or cryptography surface in this phase. The HUD performs no sim mutation (read-only consumer).
</threat_model>
