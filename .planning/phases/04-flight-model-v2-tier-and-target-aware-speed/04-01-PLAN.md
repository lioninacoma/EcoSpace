<objective>
Reshape `FlightController.UpdateSpeedEnvelope` from a single global-`MaxSpeed` clamp into a tier- and target-aware envelope, and add the read-only `Hud.ActiveTargetIndex` accessor that couples the active target to the controller.

This is the core fix for both Phase-03 UAT flight failures (in-system over-speed and the galaxy-SOI-exit dead zone), which share one root cause: a single global `_maxSpeed = 2e20` applied at every scale. The fix is a per-tier ceiling derived from `parent.SOIMeters × k` so there is never a global intergalactic value to snap to inside a star system.

Purpose: Make flight feel correct and stay usable across every scale within one auto-scaling envelope (D-47, no separate FTL mode), per locked decisions D-40 through D-45.
Output: Modified `FlightController.cs` (reshaped envelope + 2 new exports + `_hud` reference) and `Hud.cs` (one read-only accessor).
</objective>
