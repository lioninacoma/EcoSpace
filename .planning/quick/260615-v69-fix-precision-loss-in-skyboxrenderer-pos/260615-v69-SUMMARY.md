# Quick Task 260615-v69: Fix Precision Loss in SkyboxRenderer Position

## One-liner

LCA-relative hierarchy walk replaces absolute-from-root accumulation in SkyboxRenderer so the ship→body vector is computed from two small inter-body vectors rather than a difference of two large nearly-equal absolute coordinates.

## What Changed

**Scripts/Render/SkyboxRenderer.cs**

Removed: `AbsolutePositionInRoot` (private static helper that walked the full hierarchy to root and accumulated each step's metres-from-parent, producing an absolute root-frame position).

Added three new private static helpers:

| Helper | Role |
|--------|------|
| `RelativePosition(body, ship, objs, out delta)` | Orchestrator: finds LCA, accumulates each side relative to the LCA, returns `body − ship` in LCA-frame metres. Returns `false` if no common ancestor found. |
| `FindLca(a, b, objs)` | Builds the ancestor set for `a` (including `a` itself), then walks `b` upward until a common index is found. Returns −1 on failure. |
| `PositionRelativeToAncestor(node, lcaIdx, objs)` | Accumulates the node's position in metres from the LCA origin, walking child→parent and stopping when the parent index equals `lcaIdx`. Same per-step form as `AbsolutePositionInRoot` (`Units*Scale + Offset`) but terminates at the LCA, not root. |

**SyncSkyPoints** changes:
- Removed `Double3 shipRoot = AbsolutePositionInRoot(ship, objs);` (was hoisted outside the loop).
- Inside the per-body loop, replaced `bodyRoot = AbsolutePositionInRoot(body, objs); delta = bodyRoot - shipRoot;` with a single `RelativePosition(body, ship, objs, out Double3 delta)` call.
- No-common-ancestor case (returns `false`) is mapped to `dir3 = Vector3.Up` via the existing `len < 1e-30` branch (treated as `len = 0`).

All other logic (MaxStars cap, _skyDirs clear + rebuild, AngularRadius / ApparentBrightness, _sizes / _colors arrays, shader push block) is unchanged in behavior.

## Precision Rationale

**Why the old code was correct but fragile:**
At the MVP scene scale (Galaxy.LocalPos = 0, Root at origin), `AbsolutePositionInRoot` returned meters-from-origin that happened to equal meters-from-Galaxy, so `bodyRoot - shipRoot` was a subtraction of two inter-body-scale vectors — safe by coincidence.

**Why the new code is safer at arbitrary scale:**
The LCA walk accumulates each side only from the LCA downward. The subtraction is bounded by inter-body separation in the LCA frame, not by the absolute distance from root. At 1:1 Universe scale the Galaxy can be located at e.g. 1e20 m from root; the sibling stars are ~1e17 m apart. The old subtraction would be `(1e20 + 1e17) − (1e20 + 0) ≈ 1e17` — three decades of precision lost to cancellation. The new walk gives `1e17 − 0` directly.

**MVP equivalence:**
LCA(ship, sibling) = Galaxy (index 1). Galaxy.LocalPos = (0,0,0) in Universe space. Root is at the origin. So `PositionRelativeToAncestor(sibling, galaxy)` equals the old `AbsolutePositionInRoot(sibling)`, and `PositionRelativeToAncestor(ship, galaxy)` equals the old `AbsolutePositionInRoot(ship)`. The delta and all downstream values (dir3, _skyDirs, len, AngularRadius inputs, ApparentBrightness inputs) are identical within float tolerance.
