# Quick Task: Resolve phase-spanning tiered-rendering coverage gap

Documentation-only. The tiered rendering requirements (RND-02, RND-04, RND-05,
RND-07) describe behavior that is delivered incrementally: the in-system portion
lands in Phase 1/2, but the galaxy-tier portion (in-galaxy stars become meshes;
skybox carries only other galaxies; Star→Galaxy handoff) lands in **Phase 3**.
Traceability maps each to a single phase, so Phase 3 has **no traced rendering
requirement** (only TRV-02). Close that gap.

## Approach: annotate, do not split

Keep the unified tiered requirement IDs (the prior refinement deliberately
fused them into single tier rules — splitting would fragment that model).
Instead, show the phase span in traceability and give Phase 3 explicit
galaxy-tier rendering coverage.

## Tasks

### Task 1 — REQUIREMENTS traceability: mark phase-spanning reqs
- In the traceability table, expand the Phase cell for RND-02, RND-04, RND-05,
  RND-07 to name both the early (in-system) phase and Phase 3 (galaxy tier).
- Add a short note under the Coverage block explaining that tiered rendering
  requirements are delivered incrementally and counted once.

### Task 2 — ROADMAP Phase 3: add galaxy-tier rendering coverage
- Phase 3 **Requirements** line: add the galaxy-tier portions of RND-02,
  RND-04, RND-05, RND-07 alongside TRV-02.
- Add a Phase 3 **Success Criterion**: the current galaxy's stars render as
  emissive meshes (promoted from skybox points) in Galaxy space, the skybox
  there carries only other galaxies, and the promotion/demotion handoff is
  imperceptible (RND-07 at the Star→Galaxy boundary).

## Out of scope
No new requirement IDs, no renumbering, no code changes. Research-doc drift
(finding #6) remains untouched.
